Render-Fidelity Spike: Math, Code & Tables in A4

chat-to-pdf · GitHub issue #2 · browser phase · self-contained, no network assets

This document is a torture test for the on-device HTML→PDF pipeline. It mixes KaTeX-rendered mathematics, a syntax-highlighted code listing long enough to cross a page boundary, and a wide multi-row table. Everything is inlined so the exact same markup can be handed to expo-print's Android WebView print engine without a server, a build step, or a single network request.

4. Closing prose

If this page renders with correct math glyphs, monospaced and colored code that breaks cleanly across the page boundary, an unbroken bordered table, and no trailing blank page, the browser-side render approach is validated. The remaining risk lives entirely in the delta between Chromium’s print engine and Android’s WebView print engine, which the on-device phase measures next.

  • Fonts are embedded as base64 woff2 — no @font-face network fetch.
  • Layout uses display: block / table only — no flex or grid.
  • All page-break hints are expressed with widely-supported CSS 2.1 properties.
